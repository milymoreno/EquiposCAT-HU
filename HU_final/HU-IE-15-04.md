## Proceso de Resolución de Errores

1. **Detección**: El sistema identifica errores durante la sincronización o carga masiva (Ej: Incoterm faltante, proveedor no creado, referencias duplicadas).
2. **Visualización**: Se habilita una pantalla de "Log de Errores de Interfaz" donde se listan las facturas fallidas con su descripción técnica.
3. **Acciones Correctivas**:
   - **Corrección en SII**: El sistema debe permitir la edición manual de **5 campos específicos de las INV** (según definición de negocio) para evitar el re-proceso completo.
   - **Notificación Manual**: Capacidad de disparar notificaciones con estados personalizados como **"Sin Licencia"** o **"Referencia sin costo"** para alertar a los responsables de compras/logística.
   - **Re-sincronización**: Opción para volver a jalar la factura desde Dynamics una vez corregida la asociación de errores en el ERP.
   - **Eliminación y Reintento**: En casos críticos, permitir borrar la carga fallida para reintentar el proceso masivo.
4. **Flujo de Seguimiento**: El sistema marca como "Resuelto" el error una vez que la factura entra exitosamente al flujo de REQ-18.

---

## Reglas de Negocio

| ID    | Regla |
|-------|-------|
| **RN-01** | **Trazabilidad**: Todo ajuste manual en el SII sobre una factura sincronizada debe quedar registrado en un historial (quién, cuándo, qué cambió). |
| **RN-02** | **Bloqueo por Proceso**: Una factura con error no puede ser asociada a una Guía (REQ-18) hasta que sea resuelta. |
| **RN-03** | **Notificación de Error**: El sistema debe alertar proactivamente a los responsables cuando la tasa de error en una carga masiva supere un umbral definido. |
